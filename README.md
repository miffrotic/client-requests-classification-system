<h1 align="center">Система автоматической категоризации обращений в службу поддержки</h1>
<img align="center" src="https://repository-images.githubusercontent.com/1070318975/bcdc9d94-a220-47c3-ba05-e417af29846f">
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;В условиях современного бизнеса, где крупные компании ежедневно обрабатывают тысячи обращений в службу поддержки, автоматизация процессов становится ключевым фактором повышения эффективности. Проект «Автоматическая категоризация обращений в службу поддержки» направлен на решение задачи классификации и кластеризации запросов клиентов, что позволяет оптимизировать работу службы поддержки. Автоматизация категоризации тикетов по типу проблемы (например, «доступ», «сеть», «программное обеспечение», «биллинг») ускоряет маршрутизацию запросов к соответствующим специалистам, минимизирует время ожидания клиентов и снижает нагрузку на персонал. Кроме того, проект включает разработку интуитивно понятного интерфейса для взаимодействия с системой, что делает её удобной для использования в реальных условиях.

<br>

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;**Автоматическая категоризация обращений в службу поддержки** — интеллектуальная система, предназначенная для анализа, классификации и кластеризации тикетов, поступающих в службу поддержки крупных компаний. Проект реализует современные подходы машинного обучения и обработки естественного языка (NLP) для автоматизации рутинных процессов и повышения эффективности работы специалистов.

## Цели и задачи проекта

### Цели

- Автоматизировать процесс классификации тикетов по заранее заданным категориям.

- Обеспечить возможность кластеризации новых или неклассифицированных тикетов для выявления новых категорий.

- Предоставить интерпретируемые результаты работы модели для повышения доверия к системе.

- Разработать удобный интерфейс (API, веб-приложение или Telegram-бот) для интеграции с корпоративными системами и взаимодействия с пользователями.

### ML-задачи

- **Классификация тикетов:**\
Разработка модели машинного обучения для автоматического определения категории обращения на основе текста (например, «доступ», «сеть», «ПО», «биллинг»).

- **Кластеризация «неизвестных» тикетов:**\
Выделение новых категорий для тикетов, которые не подходят под существующие классы, с использованием методов кластеризации.

- **Интерпретация модели:**\
Использование методов интерпретации (SHAP, LIME) для объяснения решений модели, что повышает прозрачность и доверие к системе.

### Сервисная часть

- **API:**\
Разработка API для интеграции с корпоративными системами (например, почтой или системой тикетов, таких как Jira, Zendesk)

- **Интерфейс:**\
Telegram-бота, который принимает текст обращения, возвращает категорию, вероятность принадлежности к категории и список похожих запросов.

### Этапы выполнения проекта

<hr>

**Этап 1: Сбор и подготовка данных**

***Цель:*** Сформировать качественный датасет для обучения и тестирования моделей.

***Действия:***
- Собрать исторические данные обращений в службу поддержки (например, из системы тикетов или корпоративной почты).

- Провести очистку данных: удаление дубликатов, обработка пропусков, устранение шумов (например, орфографических ошибок).

- Выполнить разметку данных (если требуется) для создания набора с метками категорий (например, «доступ», «сеть», «ПО»).

- Разделить данные на тренировочную, валидационную и тестовую выборки.

***Инструменты:*** Python, pandas, BeautifulSoup, Selenium.

***Результат:*** Подготовленный датасет с текстами обращений и соответствующими категориями.

<hr>

**Этап 2: Исследование и предобработка текстов**

***Цель:*** Преобразовать текстовые данные в формат, подходящий для машинного обучения.

***Действия:***
- Провести токенизацию, лемматизацию и удаление стоп-слов.

- Применить методы представления текста: TF-IDF, word2vec, sentence embeddings (например, BERT или SBERT).

- Исследовать распределение категорий в данных для выявления дисбаланса классов.

- Применить методы аугментации данных (если необходимо) для увеличения объема датасета.

***Инструменты:*** NLTK, spaCy, transformers, scikit-learn.

***Результат:*** Векторизованные текстовые данные, готовые для обучения моделей.

<hr>

**Этап 3: Разработка и обучение ML-моделей**

***Цель:*** Создать и протестировать модели для классификации и кластеризации тикетов.

***Действия:***
- Классификация: Обучить несколько моделей (например, логистическая регрессия, SVM, случайный лес, нейронные сети на базе LSTM или трансформеров) и выбрать лучшую по метрикам (accuracy, F1-score).

- Кластеризация: Применить алгоритмы кластеризации (например, K-Means, DBSCAN) для выделения новых категорий из неклассифицированных тикетов.

- Интерпретация: Использовать SHAP или LIME для анализа влияния слов и фраз на предсказания модели.

***Инструменты:*** scikit-learn, tensorflow/pytorch, SHAP, LIME.

***Результат:*** Обученные модели классификации и кластеризации, а также интерпретируемые результаты предсказаний.

<hr>

**Этап 4: Разработка сервисной части**

***Цель:*** Создать инфраструктуру для интеграции модели в реальные процессы.

***Действия:***
- Разработать REST API с использованием FastAPI для обработки текстовых запросов и возврата предсказаний.

- Создать Telegram-бот, который принимает текст обращения, отображает категорию, вероятность и список похожих запросов.

- Настроить интеграцию с корпоративными системами (почта, тикет-системы).

***Инструменты:*** FastAPI, Telebot.

***Результат:*** Рабочий прототип API и пользовательского интерфейса.

<hr>

**Этап 5: Тестирование и оптимизация**

***Цель:*** Проверить работоспособность системы и улучшить её производительность.

***Действия:***
- Провести тестирование API и интерфейса на реальных данных.

- Оценить производительность моделей на тестовой выборке.

- Оптимизировать гиперпараметры моделей и устранить возможные узкие места (например, медленное время ответа API).

- Провести пользовательское тестирование интерфейса (Telegram-бот).

***Инструменты:*** pytest, Postman.

***Результат:*** Стабильная и оптимизированная система, готовая к внедрению.

<hr>

**Этап 6: Внедрение и поддержка**

***Цель:*** Интегрировать систему в рабочие процессы и обеспечить её стабильную работу.

***Действия:***

- Развернуть систему на сервере или в облаке (например, VPS, Azure).

- Настроить мониторинг и логирование для отслеживания работы системы.

- Обучить сотрудников службы поддержки работе с системой.

- Составить документацию по использованию API и интерфейса.

***Инструменты:*** Docker, Kubernetes, VPS/Azure.

***Результат:*** Полностью внедрённая система с документацией и поддержкой.
